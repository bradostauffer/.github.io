3 Program #1: Binary tree succinct decoding

3.1 Introduction

Serialization is the process of converting an object into a stream of bytes, in order to store the object in a file for example. With linear data structures, such as vectors or lists, the serialization is easy because each item can simply be written in their order. Reconstructing the object from its serial representation is then a matter of reading the stored sequence in order.

Because trees are not linear structures, their serialization is more difficult. For example, writing the preorder traversal of a tree in a file does not give any information about its structure.

Using a succinct encoding, it is possible to serialize a binary tree (both structure and data) with the minimal amount of information.

One simple succinct encoding is to traverse the binary tree in preorder and to output ‘1’ for an internal node and ‘0’ for a NIL leaf. The data associated to each node can be store separately in preorder as well.

Example:

For this binary tree, the succinct representation would be:

structure: 1 1 1 0 0 1 0 0 1 0 1 0 0
data: 28 47 5 23 99 8

The pseudo-code algorithm for the encoding phase is:

function EncodeSuccinctBinaryTree(Node n, String struct, String data) {
    if (n == NIL) then
        append 0 to struct
    else
        append 1 to struct
        append n.data to data
        EncodeSuccinctBinaryTree(n.left, struct, data)
        EncodeSuccinctBinaryTree(n.right, struct, data)
}

3.2 Goal

Write a program named bt_succinct_dec.cc that reads an encoded binary tree from a file, reconstructs it, and prints it in its preorder and inorder forms.

The input file contains two lines. The first line contains the structure, and is composed of sequence of ’0’s and ’1’s. The second line contains the data associated to the preorder sequence of nodes.

Any errors should be reported on the error stream, and the program should stop with an error code of 1.

$ cat bt.dat 
1 1 1 0 0 1 0 0 1 0 1 0 0
10 20 40 50 30 70
$ ./bt_succinct_dec bt.dat 
Preorder: 10 20 40 50 30 70 
Inorder: 40 20 50 10 30 70 
$ echo $?
0
$ 
$ cat bt_err1.dat 
1 1 1 0 0 1 8 0 1 0 1 0 0
10 20 40 50 30 70
$ ./bt_succinct_dec bt_err1.dat 
Error: structure line is invalid
$ cat bt_err2.dat 
1 1 1 0 0 1 0 0 1 0 1 0 0
10 20 40 50 30
$ ./bt_succinct_dec bt_err2.dat 
Error: data line is invalid
$ cat bt_err3.dat 
1 1 1 0 0 1 0 0 1 0 1 0 0
$ ./bt_succinct_dec bt_err3.dat 
Error: cannot read data line
$ cat bt_err4.dat 
$ ./bt_succinct_dec bt_err4.dat 
Error: cannot read structure line
$ ./bt_succinct_dec
Usage: ./bt_succinct_dec <input_file>
$ echo $?
1
$

4 Program #2: Binary search tree

4.1 Introduction

In lecture, we have studied the most common methods associated to binary search trees: contains(), min(), max(), insert() and remove().

Other methods are also quite common. For example:

  • floor(key): returns the largest key that is smaller or equal to the given key.
  • ceil(key): returns the smallest key that is greater or equal to the given key.
  • kth_small(kth): returns the kth smallest key.

4.2 Goal

Complete header file /home/cs36c/public/p3/bst.h with the implementation of floor(), ceil() and kth_small().

Your implementation should handle errors properly, by throwing exceptions when necessary.

A simple tester is provided in /home/cs36c/public/p3/bst_tester.cc. It builds a binary search tree out of a vector of values, and calls the methods with various parameters. You should complete this file to further test your implementation.
